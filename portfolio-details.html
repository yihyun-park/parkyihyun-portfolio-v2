분산 콘텐츠를
단일 채널로.

BRIEF

분산은 표면,
발화 일관성이 본질.

진짜 문제는 '블로그가 없다'가 아니었습니다. 채용·문화·기술 콘텐츠가 5+ 페이지에 흩어져 — 같은 회사의 이야기가 매번 다른 톤으로 발화되고 있었습니다. 독자가 쏘카를 인지하는 단위가 페이지마다 끊기는 구조였고, 메인에서 콘텐츠로 가는 진입점도 부재했습니다.

  • 발화 일관성 결손
  • 채널 부재
  • 메인 진입 단절
DECISIONS

새로 만들지 않는다,
가장 비싼 자산을 상속한다.

새 채널을 위해 별도 비주얼 언어를 만드는 안과 모채널 시스템을 그대로 상속하는 안을 비교했습니다. 전자는 정체성이 또렷해지지만 구축·운영·디자인 부채가 동시에 늘어납니다. 이미 사용자에게 학습된 자산을 다시 만들 이유가 약하다고 판단해 시스템은 100% 상속하고, 차별점은 '진입 동선'과 '카드 위계' 두 레이어에만 집중했습니다. 5개 카테고리는 햄버거가 아닌 상단 고정 탭으로 — 탐색을 한 번의 시선 안에 끝내기 위함입니다.

  • 시스템 100% 상속
  • 구축·운영 부채 최소화
  • 고정 탭 = 한 시선 탐색
  • 메인 진입 영역 신설
DELIVERABLES

화면이 아니라
'운영이 흔들리지 않는 골격'을 남긴다.

최종 산출물의 가치는 디자인된 3개 화면이 아닌, 카테고리가 5→10→20으로 늘어도 위계가 무너지지 않는 카드 규칙에 있습니다. 운영팀이 디자인에 다시 의존하지 않고도 새 글을 올릴 수 있도록 카드/리스트 위계, 반응형 분기, 메인 진입 영역 컴포넌트를 1세트로 묶어 넘겼습니다.

  • 운영 자율성 확보
  • 스케일 변화에 강한 카드 규칙
  • 진입 영역 컴포넌트
  • 반응형 분기 가이드

PC·모바일을
같은 손맛으로.

BRIEF

'다 보이는' 설계가
우선순위 판단을 사용자에게 떠넘긴 상태.

한 화면에 지표를 모두 펼친 결과, 관제 운영자가 어떤 알람부터 처리할지를 시각이 아닌 경험에 의존하고 있었습니다. '보기 좋은' 문제가 아니라 — 교대·신규 담당자가 투입되는 순간 대응 속도가 떨어지는 운영 리스크입니다. 더군다나 모바일에서는 지도·데이터 모두 소형화되어 현장 출동 중에는 쓸 수 없는 수준이었고, 보수적 공공 톤의 컬러는 정상·주의·경계를 한눈에 감별해야 하는 관제 조건과 상충했습니다.

  • 우선순위 부재
  • 모바일 사실상 불가
  • 식별성 아래 조건
DECISIONS

PC를 줄이지 않고,
모바일의 핵심을 PC로 확장.

일반적인 반응형 접근은 'PC를 덜어 모바일로 만든다'입니다. 하지만 관제는 현장 이동 중 한 손으로 올라온 알람을 처리하는 상황이 접점이라고 판단해, 모바일을 먼저 설계하고 PC를 '확장된 모바일'로 만드는 역방향 답을 선택했습니다. 모바일은 지도가 풀스크린, 데이터는 바텀시트로 떠오르는 구조 — 한 손 흐름을 우선했습니다. 보수적 톤 유지도 고려했지만, 정상·주의·경계·심각 4단을 0.3초 안에 구분하는 게 더 중요하다고 판단해 #605BFF를 도입했습니다. 식별성을 위한 트레이드오프입니다.

  • 모바일 우선 후 PC 확장
  • 지도 풀스크린 + 데이터 시트
  • 4단 상태 컬러 토큰화
  • #605BFF
DELIVERABLES

상태 컬러를 '토큰'으로 둬,
차트·카드·마커가 같은 소스로 움직이게.

대시보드·지도·관측소 상세·알림 4개 메인 뷰의 PC/모바일 디자인, 카드·차트·지도 마커 컴포넌트, 4단 상태 컬러 시스템을 산출했습니다. 핵심은 상태 컬러를 디자인 토큰으로 정의해 둔 점입니다 — 이후 새로운 지표·차트·마커가 추가되어도 같은 시각 규칙이 자동으로 적용되고, PC·모바일이 같은 소스에서 움직입니다. 시각 일관성이 운영팀의 추가 의사결정 없이 유지됩니다.

  • 4 메인 뷰 × 2 디바이스
  • 상태 컬러 토큰화
  • 카드·차트·마커 공유
  • 반응형 가이드

    TO-BE모바일 지도 전체화면 + 데이터 시트

    TO-BE메인 컬러 #605BFF + 신호등 상태 체계

상속과 분리를
같은 캔버스에서.

BRIEF

'한 메뉴' 안에 있을 때
쟃는 고유 정체성.

ESG는 모기관 산하의 장기 의제이면서도 자세와 이해관계가 다른 독립된 활동 단위입니다. 하지만 기존엔 모기관 사이트의 한 메뉴로 묷혀 있어, 장기 섐일이 공지·뉴스 타임라인에 파묻혀 노출되지 않고, 파트너 기관과의 접점도 공식 채널로 자리잡지 못했습니다. 정체성 부재는 랭앤딩 문제가 아니라 장기 의제의 가시성 이슈였습니다.

  • 장기 의제 가시성
  • 이해관계 분리
  • 독립 도메인 부재
DECISIONS

가장 비싼 자산은 공유,
가장 값싼 레이어만 분리.

'완전 분리'와 '완전 상속' 양극단은 모두 하자가 명확했습니다. 전자는 정체성은 살아도 두 번 운영해야 하는 비용을 운영팀이 짐, 후자는 채널을 만들고도 정체성이 사라집니다. 그래서 '비싼 자산'과 '값싼 자산'을 구분해 레이어별로 설계했습니다. GNB·푸터·뉴스 카드얲 운영 관리 비용이 큰 공통 자산은 100% 상속해 운영 부화를 줄이고, 컬러·아이콘·일러스트 두 레이어만 분리해 ESG만의 톤을 만들었습니다. 두 사이트는 1,600px 캔버스를 공유해 해더·푸터 컴포넌트가 그대로 디포지도록 맞춰둔 점도 같은 의도입니다.

  • 공통 자산 100% 상속
  • 컬러·아이콘 레이어만 분리
  • 1,600px 캔버스 공유
  • 컴포넌트 디포지
DELIVERABLES

디자이너 없이도
운영팀이 포장할 수 있는 템플릿.

5개 화면 디자인, ESG 전용 일러스트/아이콘, 상호 진입 컴포넌트, 컴포넌트 템플릿 라이브러리를 산출했습니다. 관점의 핵심은 '으로 설계'가 아니라 '운영 단계의 자동화'입니다. 자주 쓰이는 섹션을 템플릿으로 묶어, 새 ESG 활동 페이지를 올릴 때 디자이너 수배업 없이 운영팀이 찍어낼 수 있도록 했습니다.

  • 5 화면
  • ESG 아이콘 세트
  • 상호 진입 컴포넌트
  • 운영 자율 템플릿

상속과 분리를

    TO-BE공통 요소 상속 + ESG 전용 컬러·아이콘

PC 전용을 벗어나,
같은 그리드로 어디서든.

BRIEF

'구식'이 아니라
주 사용자에게 닫힌 구조.

공공기관 사이트의 주 사용자는 중장년이지만, 기존 사이트는 본문 글자 12–13px에 PC 전용으로 구축돼 대상 사용자에게는 사실상 접근이 차단된 구조였습니다. 더군다나 공시·자료실·사업의 IA 깊이가 3단 이상이고 메인 탐색이 드롭다운 메뉴에 의존해, 필요한 정보까지 닿으려면 4회 클릭이 필요했습니다. 단순한 '최신화' 문제가 아닌 접근성의 문제로 재정의했습니다.

  • 대상 사용자에게 닫힌 구조
  • 4 클릭 깊이
  • PC 전용 구축
DECISIONS

모바일 재설계가 아니라,
PC·모바일을 '하나의 원칙'으로.

흔한 공공 리뉴얼은 모바일을 '추가 타깃'으로 재설계합니다. 하지만 운영팀이 하나의 소스로 두 면을 관리해야 하는 조건을 고려해, 내부 IA·컴포넌트를 동일하게 둔 채 그리드만 달리하는 단일 원칙 설계로 갈래했습니다. 본문 글자는 16px을 베이스로 — '단순히 크게'가 아니라 공공 사이트의 가독성 기준선을 수립하는 단위입니다. 좌측 고정 GNB는 햄버거의 '숨김'을 거부하고 깊은 IA를 수직으로 곧바로 노출해 클릭 깊이를 줄이는 판단이었습니다 — 탐색에 필요한 인지 부하를 시각에 넘긴 설계입니다.

  • PC·모바일 동일 원칙
  • 본문 16px = 가독성 기준선
  • 좌측 고정 GNB · 클릭 깊이 감소
  • 1,600 그리드
DELIVERABLES

'메뉴가 늘어도 한 화면에 도달'
운영 확장성까지 고려한 설계.

메인·소개·사업·자료실·문의 5개 화면, 좌측 고정 GNB 컴포넌트, 뉴스/보도자료 카드 시스템, 반응형 가이드를 산출했습니다. 고정 GNB는 단순 도식이 아니라 운영 단계에서 메뉴가 N개 추가돼도 곧바로 한 화면에서 접근되도록 스크롤 규칙을 내장한 설계입니다. 조직 개편·사업 확대로 IA가 팽창해도 탐색 구조가 무너지지 않습니다.

  • 5 화면
  • 고정 GNB(스크롤 내장)
  • 뉴스 카드 시스템
  • 반응형 가이드
NOTE 본 산출물은 설계 단계에서의 본인 의사결정 기반 디자인입니다. 실제 운영 디자인은 이후 외부 카피 방향으로 전환되어, 본 페이지에 담은 산출물과는 다를 수 있음을 밝힙니다.

채널이 늘어날수록 톤이 흔들리던 홍보물을 — 시스템과 채널 그리드를 다른 레이어로 분리해 해결한 프레임워크

KEYWORD
콘텐츠 · 홍보 디자인
DATE
2023 — 2025
SCOPE
UI · 콘텐츠 디자인 단독
ROLE

(STORY · 01 → 02 → 03)

채널은 분리,
톤은 하나로.

BRIEF

콘텐츠가 아니라
'프레임워크'가 없었다.

카드뉴스·EDM·배너 등 채널별 홍보물에 일관된 가이드가 없어 같은 메시지가 채널마다 다르게 보이고 있었습니다. 그러나 진짜 비용은 콘텐츠가 채널을 건널 때마다 다시 디자인되는 운영 부하에 있습니다. 단일 캠페인이 EDM·카드뉴스·배너로 퍼질 때마다 세 번의 디자인 결정이 필요했고, 시스템 컬러·타이포와도 분리되는 톤 균열이 누적되면서 브랜드 일관성까지 흐릿해졌습니다.

  • 프레임워크 부재
  • 채널별 디자인 재작업
  • 시스템과 톤 분리
DECISIONS

'하나의 가이드'와 '채널별 개별 제작'
의 중간해.

하나의 가이드로 묶으면 채널 특성(EDM의 롱폼 구조, 카드뉴스의 스와이프, 배너의 고정 프레이밍)이 죽고, 채널별로 완전 분리하면 톤 균열이 재발합니다. 두 극단 대신 시스템 레이어(컬러·타이포·CTA)는 공유하고, 채널 레이어(그리드·비율·이미지 운용)만 따로 정의하는 절충해를 선택했습니다. 더 중요한 건, 메시지를 헤드라인 → 서브 → CTA 3단 골격으로 강제해 어떤 채널을 타도 핵심 메시지가 같은 위계에 도달하게 만든 대목입니다.

  • 시스템 × 채널 레이어 분리
  • 공유: 컬러·타이포·CTA
  • 분리: 그리드·비율
  • 3단 골격 강제
DELIVERABLES

마스터 템플릿 —
새 캠페인의 비주얼 결정 비용을 0으로.

카드뉴스 템플릿(정사각·세로형), EDM 마스터 템플릿, 사이즈별 배너 가이드(웹·SNS), 컬러·타이포·CTA 컴포넌트 사용 규칙을 정리해 산출했습니다. 핵심은 마스터 템플릿이 새 캠페인이 시작될 때마다 운영팀이 지는 비주얼 결정 비용을 0에 가깝게 만든다는 점입니다. 디자이너 개입 없이도 일관된 출력이 가능해졌습니다.

  • 카드뉴스 2종
  • EDM 마스터
  • 배너 가이드
  • 운영 자율 프레임워크

네이버페이 × 탄소중립포인트

복잡한 신청 절차가 일반 사용자에게 진입 장벽이었던 캠페인 — 정보 흐름을 모바일 1단으로 다시 짠 랜딩

KEYWORD
공공 캠페인 · 랜딩
DATE
2023

UI 디자인 + 반응형 퍼블리싱 단독
ROLE

(STORY · 01 → 02 → 03)

복잡한 절차를
한 손에 흐르도록.

BRIEF

'복잡함'이 아니라
'예측 불가능'이 진입 장벽.

탄소중립포인트는 전환 절차가 복잡해 일반 사용자에게 진입 장벽이 높은 캠페인이었습니다. 그러나 더 정확한 진단은 단계가 많다는 사실 자체가 아니라, 단계가 시각화되지 않아 '얼마나 남았는지'를 가늠할 수 없다는 점이었습니다. 사용자는 다음 화면을 예측하지 못한 상태로 절차를 시작했고, 가장 중요한 CTA마저 페이지 깊은 곳에 묻혀 결심 시점을 놓치는 구조였습니다.

  • 예측 불가능
  • 단계 시각화 부재
  • CTA 가시성 낮음
DECISIONS

설명이 아니라 '예측'을 준다 —
실 UI 캡처 + 모바일 1단.

모바일 기준 세로 1단 레이아웃으로 정보 흐름을 — 제도 소개 → 참여 방법 → 단계 가이드 → CTA의 4단으로 단순화했습니다. 핵심 의사결정은 텍스트 가이드 대신 실 UI 캡처를 가공해 단계마다 보여준 점입니다. 글로 설명하는 것보다 다음 화면을 사용자가 직접 예측하게 만드는 편이 이탈을 줄이는 데 결정적이라는 판단이었습니다. CTA는 상·하단 두 곳에 배치 — 단순한 패턴이 아니라, 모바일 세로 1단에서는 사용자의 '결심 시점'이 두 번 발생한다는 점을 의도한 설계입니다.

  • 실 UI 캡처 → 예측 가능성
  • 모바일 세로 1단
  • 상·하단 CTA = 결심 시점 2회
  • 4단 흐름
DELIVERABLES

디자인-퍼블리싱 1인 단독,
의사결정 전달 비용 0.

메인 랜딩 페이지의 디자인과 반응형 퍼블리싱을 1인이 단독으로 진행했습니다. 분리된 역할이었다면 '의도'가 코드로 옮겨질 때 발생하는 손실이 있지만, 디자인-퍼블리싱을 한 사람이 끝까지 책임지면서 의사결정의 정확도와 마감 속도를 동시에 잡았습니다. 단계 가이드 컴포넌트, 실 UI 캡처 가공물, N마크 + 일러스트 시각 자산을 포함합니다.

  • 1 페이지
  • 4단계 가이드
  • 반응형 퍼블리싱
  • 의도 손실 0
